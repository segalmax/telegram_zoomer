## 9. Translation Memory (ðŸ§  Supabase + pgvector)
**Purpose**: keep the bot consistent by remembering past `source â†’ translation` pairs and feeding semantically-similar examples into every new translation request.

### How it works
1. **Embed** â€“ original Russian text â†’ 1536-d vector via OpenAI `text-embedding-ada-002` (configurable via `EMBED_MODEL`).
2. **Store** â€“ row inserted/updated in Supabase table `article_chunks`:
   | column | type | note |
   |--------|------|------|
   | `id` | text PK | `<msgId>-right` etc. |
   | `source_text` | text | original post |
   | `translation_text` | text | zoomer output |
   | `embedding` | vector(1536) | pgvector extension |
   | `created_at` | timestamptz | UTC |
3. **Recall** â€“ before translating a new post the bot calls`match_article_chunks(query_embedding, k)` (SQL function) to fetch the **top k (default 5)** closest matches by cosine similarity.
4. **Context inject** â€“ these matches are prepended to the prompt so Claude keeps terminology & tone consistent.

### Setup (already automated)

## 10. Session Management - Simplified Interactive Approach (updated 2025-01-14)

**Philosophy**: No session transfers, no compression, no environment variables. Just clean interactive sessions.

### Session Strategy
1. **Local Development**
   â€¢ Uses `session/local_bot_session.session`
   â€¢ Created interactively on first run (prompts for phone/code)

2. **Heroku Production** 
   â€¢ Uses `session/heroku_bot_session.session`
   â€¢ Created interactively on first Heroku run (prompts for phone/code)
   â€¢ No session transfers from local to Heroku

3. **Testing**
   â€¢ Uses `session/sender_test_session.session`
   â€¢ Created interactively on first test run (prompts for phone/code)
   â€¢ Separate test account recommended to avoid conflicts

### Deployment Process
```bash
# Deploy to Heroku (no session handling)
./setup_heroku.sh

# First run on Heroku will prompt for authentication
heroku logs --tail --app your-app-name
```

### Benefits
- **No session corruption** from transfers/compression
- **Clean separation** between environments  
- **Interactive authentication** ensures sessions work properly
- **No complex environment variables** for session management
- **Easier debugging** - sessions are just local files

### Session Files
- `session/local_bot_session.session` - Local development
- `session/heroku_bot_session.session` - Heroku production (if exists locally)
- `session/sender_test_session.session` - Testing
- `session/app_state.json` - Application state (local only)
